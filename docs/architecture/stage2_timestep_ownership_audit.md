# Stage 2 timestep ownership audit

Date: 2026-05-10  
Scope: infrastructure audit only; no solver math, restart schema, or runtime behavior changes.

## Purpose

Stage 2 timestep repair is blocked until every timestep-related state lane has one explicit owner. This audit maps the criteria-to-scheduler path, scheduler legality, bin commit, activation, integration, mirror update, restart/provenance serialization, and structural-transform preservation path so later patches do not create split-brain timestep truth.

## Audit sweep

The audit used repository-wide `rg` searches for at least the following terms: `time_bin`, `TimeBin`, `timestep`, `time_step`, `dt_time_code`, `current_tick`, `next_activation_tick`, `active_flag`, `pending_bin_index`, `active_set`, `ActiveSetDescriptor`, `scheduler`, `rung`, `bin`, `kick`, `drift`, `cadence`, `gravity_kick_opportunity`, `pm_update_cadence_steps`, `restart`, `checkpoint`, `migration`, `reorder`, and `compaction`.

Inspected code surfaces include:

- Core timestep API and implementation: `include/cosmosim/core/time_integration.hpp`, `src/core/time_integration.cpp`.
- State lanes and structural transforms: `include/cosmosim/core/simulation_state.hpp`, `src/core/simulation_state.cpp`, `src/core/simulation_state_species.cpp`, `src/core/simulation_state_structures.cpp`.
- Workflow runtime path: `src/workflows/reference_workflow.cpp`.
- Restart and snapshot I/O: `include/cosmosim/io/restart_checkpoint.hpp`, `src/io/restart_checkpoint.cpp`, `src/io/snapshot_hdf5.cpp`.
- Distributed cadence and restart compatibility: `include/cosmosim/parallel/distributed_memory.hpp`, `src/parallel/distributed_memory.cpp`.
- Timestep, restart, reorder/compaction, migration, PM cadence, and workflow tests under `tests/unit` and `tests/integration`.

## Ownership model for Stage 2

`core::HierarchicalTimeBinScheduler` is the intended owner for discrete timestep truth: element bin assignment, legal transition timing, activation tick, active flag for the current substep, pending bin transition, and scheduler current tick. All state `time_bin` arrays are mirrors derived from scheduler persistent state. `core::IntegratorState` owns the global continuous step coordinates (`current_time_code`, `current_scale_factor`, `dt_time_code`, `step_index`) and stage metadata, but it does not own per-element bin truth. `src/workflows/reference_workflow.cpp` is the composition layer that turns CFL/gravity proposals into scheduler transition requests, builds active descriptors from the scheduler, executes the orchestrator, and synchronizes mirrors after bin commits.

## Mutation path trace

1. **Criteria proposal**: `updateAdaptiveTimeBins` builds `TimeStepLimits` from `IntegratorState::dt_time_code` and scheduler `maxBin`, then registers hydro CFL and gravity acceleration hooks per element. The combined criterion maps to a target bin via `mapDtToTimeBin`.
2. **Scheduler request**: workflow calls `HierarchicalTimeBinScheduler::requestBinTransition`. The request writes only `pending_bin_index`; it does not immediately mutate committed `bin_index`.
3. **Activation**: `beginSubstep()` rebuilds the active cache from `current_tick`, `bin_index`, and `next_activation_tick`. Workflow partitions scheduler element indices into active particles and active cells.
4. **Descriptor freshness**: `makeSchedulerActiveSetDescriptor` stamps active particle/cell spans with state generation counters and `source_scheduler_tick`. `StepOrchestrator` can validate expected scheduler tick before callbacks run.
5. **Integration**: drift, gravity, hydro, source, analysis, and output callbacks consume the active descriptor. They do not own bin legality. Gravity owns PM kick-opportunity cadence state because that counter advances per gravity kick stage, not per scheduler substep.
6. **Bin commit and tick advance**: `endSubstep()` applies legal pending transitions, clears current active flags, advances `next_activation_tick`, and increments `current_tick`.
7. **Mirror sync**: workflow calls `syncTimeBinsFromScheduler` after `endSubstep()` so `ParticleSoa::time_bin` and `CellSoa::time_bin` reflect scheduler `bin_index` before output/restart.
8. **Output/restart**: restart writes scheduler persistent state as authority and writes state time-bin arrays as mirrors. Snapshot paths are diagnostic and do not serialize exact scheduler continuation state.

## Unsafe hidden state and follow-up repair prompts

1. **State `time_bin` mirrors are high-risk obsolete duplicates unless every production mutation flows from scheduler sync.** Follow-up: add runtime assertions at restart write and structural-transform boundaries that mirrors match scheduler when a scheduler is in scope.
2. **Scheduler element order after reorder, compaction, and migration is not self-describing in the scheduler state.** Follow-up: require an explicit scheduler rebuild/reimport contract after any structural transform that changes local element order or count.
3. **`IntegratorState::time_bins.active_bin` is metadata without current production authority.** Follow-up: either define it as diagnostic-only or remove/replace it in a schema-versioned change; do not let callbacks read it as active-set truth.
4. **Restart reads both scheduler authority and time-bin mirrors.** Follow-up: add read-side compatibility checks that compare mirror datasets with scheduler `bin_index` before accepting exact continuation.
5. **PM cadence callback state is authoritative at runtime but serialized via workflow copying.** Follow-up: when restart continuation is implemented, rehydrate `GravityCallback` cadence state from `DistributedRestartState` before any gravity kick.
6. **Snapshot time-bin defaults can be mistaken for restart continuity.** Follow-up: document and enforce snapshot-as-analysis-input semantics for hierarchical timestep lanes.
7. **Active descriptors are transient but easy to cache accidentally.** Follow-up: keep generation/tick freshness checks in every solver callback entry point that consumes subset descriptors.

## Reproducibility impact

This audit is documentation-only and does not change deterministic behavior, output naming, normalized config dumps, restart schemas, or solver math. Its intended reproducibility impact is to prevent later Stage 2 changes from weakening exact restart continuation: scheduler persistent state remains the only per-element timestep authority, state arrays remain mirrors, and PM cadence state remains tied to gravity kick opportunities.
